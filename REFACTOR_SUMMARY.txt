✓ KEY FEATURES

  Architecture:
    - Metrics: Pull-based (Prometheus scrapes edge exporters)
    - Logs: Push-based (Promtail pushes to central Loki)
    - Central Grafana: Single pane of glass for all nodes
    - No single point of failure for edge nodes (each is independent)

  Environment Variables:
    - NODE_NAME: Unique identifier for each edge node
    - CENTRAL_HOST: Central server reachable from edges
    - LOKI_URL: Log endpoint for remote push
    - All exporter ports configurable per deployment

  Prometheus Targets:
    - Load from file_sd_configs (prometheus/targets/edge-nodes.yml)
    - Supports DNS hostnames (gpu-node-01) or IP addresses (10.0.0.21)
    - 30-second refresh interval (no restart needed to add nodes)

  Promtail Logging:
    - Labeled with node name (${NODE_NAME})
    - Pushes to central Loki endpoint (${LOKI_URL})
    - Queries in Grafana: {job="varlogs", node="gpu-node-01"}

================================================================================
  DEPLOYMENT WORKFLOW
================================================================================

CENTRAL SERVER (One instance)
  1. git clone && cd server-observability
  2. cp .env.example .env
  3. Edit .env (CENTRAL_HOST, GF_SECURITY_ADMIN_PASSWORD)
  4. docker compose -f docker-compose.central.yml up -d
  5. Access Grafana: http://<central-ip>:3000

EDGE NODE (Repeat for each GPU server)
  1. git clone && cd server-observability
  2. cp .env.example .env
  3. Edit .env (NODE_NAME=gpu-node-01, CENTRAL_HOST=10.0.0.10, LOKI_URL=...)
  4. docker compose -f docker-compose.edge.yml up -d
  5. On central: Edit prometheus/targets/edge-nodes.yml, add node
  6. curl -X POST http://central:9090/-/reload
